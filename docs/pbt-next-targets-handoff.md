## Target 1: SyncIndex Idempotence

Area: `internal/agents`

Relevant code:

- `SyncIndex(doc string, cards []IndexCard) (string, error)`
- `RenderBlock(cards []IndexCard) (string, error)`
- `RenderIndex(cards []IndexCard) (string, error)`

Property:

```txt
SyncIndex(SyncIndex(doc, cards), cards) == SyncIndex(doc, cards)
```

Intent:

After the first sync, running sync again with the same card set should produce no further changes.

Suggested generator shape:

- Generate valid `IndexCard` values using the existing `quickIndexCards` generator from `internal/agents/context_index_property_test.go`.
- Generate a small set of valid document shapes instead of arbitrary strings.
- Include documents with no Garden block.
- Include documents with an existing Garden block generated by `RenderBlock`.
- Include documents with an existing Garden block and stale index.
- Include human content before and after the managed block.

Keep invalid marker documents out of this property. Malformed marker behavior is a separate defensive-test concern.

Acceptance criteria:

- Add a property test in `internal/agents`.
- Property runs with bounded `quick.Config{MaxCount: ...}`.
- Full suite passes with `env GOCACHE=/tmp/garden-go-build go test ./...`.

## Target 2: UpsertBlock Preserves Non-Garden Content

Area: `internal/agents`

Relevant code:

- `UpsertBlock(doc string, block string) (string, error)`
- `findSingleMarkerRange(...)`
- `appendBlock(...)`

Property:

```txt
When doc contains exactly one Garden agents block,
UpsertBlock(doc, newBlock) preserves the exact prefix before the old block
and the exact suffix after the old block.
```

Intent:

Garden may replace its managed section, but it must not rewrite surrounding human-authored AGENTS.md content.

Suggested generator shape:

- Generate safe human text for prefix and suffix.
- Exclude Garden markers from generated human text.
- Avoid control characters and excessive size.
- Build `doc` as `prefix + oldBlock + suffix`.
- Use a valid `newBlock` containing `AgentsStartMarker` and `AgentsEndMarker`.

Expected checks:

```txt
strings.HasPrefix(got, prefix)
strings.HasSuffix(got, suffix)
strings.Contains(got, newBlockWithoutUnexpectedMutation)
oldBlock is absent unless oldBlock == newBlock
```

Gotcha:

`UpsertBlock` may normalize blank lines around blocks depending on the exact document shape. Keep this property focused on documents where prefix and suffix include the separators exactly as intended, or assert preservation by slicing around marker positions rather than expecting a fully unchanged surrounding whitespace policy for every arbitrary prefix/suffix.

Acceptance criteria:

- Add a property test in `internal/agents`.
- The generator must not accidentally create malformed Garden marker documents.
- Full suite passes.

## Target 3: Parse Invalid Frontmatter Fails Safely

Area: `internal/contextcard`

Relevant code:

- `Parse(path string, data []byte) (Card, error)`
- `validateSlug(slug string) error`
- `fieldLooksScalar(frontmatter string, field string) bool`
- `validateCompactIndexScope(scope []string) error`

Property:

```txt
Generated invalid card documents should never parse successfully,
should never panic,
and should return a controlled error message.
```

Intent:

Parsing context cards is a boundary. Invalid Markdown/frontmatter should fail predictably and should not create partially trusted cards.

Suggested invalid classes:

- Missing opening frontmatter marker.
- Missing closing frontmatter marker.
- Scalar `scope` instead of list.
- Empty `scope`.
- Scope containing `CHANGE_ME`.
- Scalar `tags` instead of list.
- Scope containing compact row delimiter: `|`.
- Control characters in scope.
- Empty body.
- Invalid slug in path.

Suggested generator shape:

- Generate a tagged invalid case type, not fully arbitrary YAML.
- For each generated case, render a card-like string that is invalid for exactly one primary reason when possible.
- Use valid slugs except in the invalid-slug case so failures stay localized.

Expected checks:

```txt
_, err := Parse(path, []byte(content))
err != nil
err.Error() != ""
```

Optional stronger check:

Match error substrings against the known controlled messages already asserted in `internal/contextcard/card_test.go`.

Known controlled messages:

```txt
YAML frontmatter is required
scope must include at least one glob
scope must be a list
scope cannot contain CHANGE_ME
tags must be a list
scope contains compact index syntax delimiter
scope contains compact index syntax control character
body cannot be empty
invalid card slug
```

Acceptance criteria:

- Add a property test in `internal/contextcard`.
- The test recovers from panics or relies on `quick.Check` surfacing panics as failures.
- Invalid generated cases do not accidentally become valid cards.
- Full suite passes.

## Implementation Order

1. Add `SyncIndex` idempotence first. It is the cleanest high-value property.
2. Add `Parse` invalid frontmatter next. It protects a boundary and can reuse existing example-test expectations.
3. Add `UpsertBlock` preservation last. It is valuable but needs the most careful generator design around whitespace and markers.
